Glosario

1. Servidor de aplicaciones

Un servidor de aplicaciones es un software que ayuda a un servidor web a ejecutar scripts o etiquetas del lado del servidor en páginas web. Cuando se solicita una página de este tipo al servidor, el servidor web la reenvía al servidor de aplicaciones para que la procese antes de proporcionarla al navegador. Para obtener más información, consulte Cómo funciona una aplicación web.
 
Macromedia ColdFusion, Macromedia JRun Server, Microsoft.NET Framework, IBM WebSphere y Apache Tomcat son servidores de aplicaciones típicos.

2. Base de datos

En pocas palabras, una base de datos es una colección de datos estructurados. Considera tomarte una selfie: presionas un botón para tomarte una foto. Su imagen son datos y la galería de su teléfono es la base de datos. Una base de datos es una ubicación para almacenar y organizar datos. El término "relacional" indica que los datos del conjunto de datos están estructurados en tablas. Cada tabla está conectada de alguna manera.

3. Tecnología de servidor

Una  tecnología de servidor emplea tecnologías de servidor para cambiar sitios web dinámicos durante el tiempo de ejecución.
El entorno de desarrollo de Dreamweaver es compatible con las siguientes tecnologías de servidor:

  • Macromedia Cold Fusion
  • El servidor Microsoft ASP.NET
  • Páginas activas del servidor (Active Server Pages) (ASP)
  • Páginas del servidor Sun Java (JSP)
  • Preprocesador de hipertexto (PHP) (PHP)

El entorno de codificación de Dreamweaver también se puede utilizar para crear sitios para tecnologías de servidor no especificadas.

4. Aplicación web

Una aplicación web es un sitio web que tiene páginas con contenido desconocido, ya sea parcial o totalmente. Solo cuando un visitante solicita una página del servidor web se decide el contenido final de estas páginas. Este tipo de página se conoce como página dinámica, ya que el contenido final de la página cambia de una solicitud a otra según las actividades del visitante.

5. Servidor web

En respuesta a las solicitudes de los navegadores web, el software del servidor web transmite páginas web. Se realiza una solicitud de página cuando un usuario hace clic en un enlace en una página web, elige un marcador en el navegador o ingresa una URL en la barra de direcciones del navegador.


Los servidores web populares incluyen Microsoft Internet Information Server, Microsoft Personal Web Server, Apache HTTP Server, Netscape Enterprise Server y Sun ONE Web Server.

6. Página estática

Una página estática es una página web que no ha sido alterada por un servidor de aplicaciones antes de ser transmitida a un navegador. Consulte Procesamiento de páginas web estáticas para obtener más detalles.

7. Página dinámica

Una página dinámica presenta material único para cada usuario manteniendo el mismo diseño y apariencia. Estas páginas, que a menudo se crean en CGI, AJAX, ASP o ASP.NET, tardan más en cargarse que las páginas estáticas. Por lo general, se usan para mostrar información que cambia con frecuencia, como precios de acciones o actualizaciones del clima.

Por lo general, los sitios web dinámicos incluyen programas de aplicación para varios servicios y necesitan recursos del lado del servidor, como bases de datos. Una base de datos permite al diseñador de la página separar el diseño del sitio web del contenido que se presentará a los visitantes. Una vez que han subido material a la base de datos, el sitio web lo recupera en respuesta a la solicitud de un usuario.

8. Client-side Scripting

Las secuencias de comandos del lado del cliente se utilizan para modificar una página web en respuesta a una acción del usuario ("evento del lado del cliente"). Estos scripts producen "contenido del lado del cliente" en la computadora del usuario, a diferencia del servidor web.

9. Server-side Scripting

Usando secuencias de comandos del lado del servidor, una página web que cambia a medida que se carga, se ve o, según lo que se proporciona, lo hace. El contenido del lado del servidor se crea cuando se carga la página. Las páginas de inicio de sesión, los carritos de compras y los formularios de envío son ejemplos.

10. Views(Vista)

Una vista es un invocable que acepta y entrega una respuesta a una solicitud. Esto puede ser cualquier cosa que no sea una función, y Django incluye ejemplos de clases que se pueden usar como vistas. Mediante el uso de herencia y mixins, le permiten organizar sus vistas y reutilizar el código. También hay vistas genéricas para tareas, de las que hablaremos más adelante, pero puede optar por desarrollar su propia estructura de vistas reutilizables que se adapte a su caso de uso.

11. Models(Modelos)

Un modelo es la única fuente definitiva de conocimiento relacionado con los datos. Incluye los campos y comportamientos más importantes de los datos que está guardando. Cada modelo normalmente corresponde a una sola tabla de base de datos.

  • Cada modelo es una clase de Python que es una subclase de django.db.models.Model
  • Cada atributo del modelo corresponde a un campo de la base de datos.
  • Como resultado, Django proporciona una API de acceso a la base de datos construida automáticamente.

12. Model–view–controller (MVC)

Es un diseño de arquitectura de software de uso frecuente para construir interfaces de usuario que divide la lógica del programa relevante en tres piezas interrelacionadas. Esto se hace para distinguir las representaciones internas de la información de los métodos a través de los cuales el usuario recibe y acepta la información.


Este estilo, que se usaba tradicionalmente para las interfaces gráficas de usuario (GUI) de escritorio, se hizo popular para crear aplicaciones en línea. Los marcos MVC para lenguajes de programación populares ayudan a la implementación del paradigma.

13. Property(atributos controlados)

También conocida como "atributos controlados", esta función de Python se introdujo en la versión 2.2. Esta es una técnica inteligente para construir atributos cuyo uso imita el acceso a atributos, pero cuya implementación se logra mediante llamadas a métodos.

14. Queryset

Una representación de las filas que se van a recuperar de la base de datos.

16. Template

Como marco web, Django requiere un método simple para generar HTML dinámicamente. El método más utilizado depende de las plantillas. Una plantilla incluye tanto las partes estáticas de la salida HTML deseada como una gramática que indica cómo se incluirá la información dinámica.

17. Localhost

El nombre de red utilizado para crear una conexión a un servidor de base de datos. Frecuentemente mencionado junto a un puerto. En algunas situaciones, la dirección IP 127.0.0.1 es preferible al nombre exclusivo localhost para acceder a una base de datos ubicada en el mismo servidor que la aplicación.

18. Python

Python es un lenguaje de programación de alto nivel orientado a objetos con semántica dinámica que es interpretable. Sus estructuras de datos integradas de alto nivel, junto con la escritura dinámica y el enlace dinámico, lo convierten en un lenguaje muy deseable para el desarrollo rápido de aplicaciones, así como para su uso como lenguaje de secuencias de comandos o pegamento para vincular componentes existentes. La legibilidad enfatizada por la sintaxis directa y fácil de aprender de Python reduce el costo de mantenimiento del software. Python permite módulos y paquetes, lo que promueve la modularidad y la reutilización de los programas. El intérprete de Python y la biblioteca estándar sustancial están disponibles gratuitamente en formato fuente o binario para todas las plataformas principales y pueden distribuirse sin restricciones.

19. MySQL

MySQL es una solución de gestión de código abierto para bases de datos relacionales (DBMS). Oracle Corporation lo desarrolla, mantiene y distribuye, y lo rodea una sólida comunidad.

20. API

Las API son los métodos que permiten que dos componentes de software interactúen utilizando un conjunto de definiciones y protocolos. Por ejemplo, el sistema de software de la oficina meteorológica almacena datos meteorológicos diarios. La aplicación meteorológica de su teléfono se "comunica" con este sistema a través de las API para mostrar actualizaciones meteorológicas diarias.
