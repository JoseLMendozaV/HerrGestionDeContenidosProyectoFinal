Django

Django es un marco de desarrollo web de código abierto basado en Python que sigue el paradigma arquitectónico modelo-vista-controlador (MVC). Primero se diseñó para operar sitios web orientados a noticias para World Company of Lawrence, Kansas, y se distribuyó al público bajo una licencia BSD en julio de 2005; lleva el nombre del músico de jazz gitano Django Reinhardt.


El objetivo principal de Django es facilitar la creación de sitios web complicados. Django enfatiza la reutilización, la conexión y extensión de componentes, el desarrollo rápido y la filosofía "Don't Repeat Yourself" (DRY). Python se usa en todos los componentes del marco, incluidas las configuraciones, los archivos y sus modelos de datos.

Componentes

A pesar de tener su propia terminología, como llamar "vistas" a los objetos que generan respuestas HTTP, el marco fundamental de Django puede verse como un diseño MVC. Consiste en un mapeador relacional de objetos (ORM) que actúa como mediador entre los modelos de datos de Python y una base de datos relacional ("Modelo"), un sistema para procesar solicitudes HTTP con un sistema de plantillas web ("Ver") y una expresión regular. despachador de URL basado ("Ver") ("Controlador").


También se incluyen en la estructura fundamental:

  • Un servidor web ligero e independiente para desarrollo y pruebas
  • Un sistema de serialización y validación de formularios capaz de traducir entre formularios HTML y valores almacenables en bases de datos.
  • un sistema de plantilla que utiliza la idea de herencia de la programación orientada a objetos
  • Un marco de almacenamiento en caché que admite varias técnicas de caché
  • Soporte para clases de middleware que pueden intervenir en diferentes etapas de procesamiento de solicitudes y ejecutar métodos personalizados.
  • Un sistema de despacho interno que permite que los componentes de la aplicación comuniquen eventos usando señales predefinidas.
  • un mecanismo para la internacionalización que incluye traducciones de los propios componentes de Django a muchos idiomas
  • Un sistema capaz de producir y leer representaciones XML y/o JSON de objetos modelo de Django.
  • Un sistema para mejorar las capacidades del motor de plantillas
  • una interfaz para el marco de prueba de unidad nativo de Python
